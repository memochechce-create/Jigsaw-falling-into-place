Разделение построено так, что один участник работает с внутренними бизнес-данными (профили пользователей и покупки с сайта бренда), а второй — с внешними рыночными данными (парсинг сторонних площадок и публичных API).

Задача А3: Вычисление бизнес-метрик по каждому пользователю
На основе профилей пользователей и их заказов участница А реализует класс UserAnalytics, который вычисляет ключевые бизнес-метрики для каждого пользователя. LTV (Lifetime Value) — суммарная выручка с одного пользователя за всё время, позволяет выявить наиболее ценных клиентов. AOV (Average Order Value) — средний чек, используется для ценового позиционирования и оценки эффективности промоакций. Churn — флаг, выставляемый пользователю, который не совершал покупок более 90 дней; применяется для выявления клиентов, которых нужно вернуть. Retention показывает, какой процент пользователей вернулся за 30, 60 или 90 дней после первой покупки, отражая лояльность аудитории. Каждому пользователю также присваивается RFM-сегмент (Champion, Loyal, Promising, At Risk или Lost) на основе давности, частоты и суммы покупок. Дополнительно вычисляются любимая тематика пазлов и предпочтительный уровень сложности (по среднему количеству деталей в заказах).

Задача А4: Бизнес-дашборд — агрегированные метрики по всей базе
Финальная задача участницы А — реализовать класс BusinessDashboard, который принимает полный список профилей пользователей и считает метрики на уровне всей компании: общая выручка, средний LTV, уровень оттока (churn rate), разбивка пользователей по RFM-сегментам, топ тематик пазлов по популярности, выручка по городам и регионам. Эти агрегированные данные визуализируются с помощью matplotlib и seaborn: столбчатые диаграммы по сегментам, тепловая карта активности покупок по месяцам (сезонность), распределение LTV по возрастным группам. Этот блок представляет собой итоговый отчёт для руководителя бизнеса, отвечающий на вопрос: как чувствует себя бренд в целом.

Участник Б — Внешние рыночные данные и анализ трендов

Задача Б1: Сбор данных о пазлах с внешних торговых площадок
Участник Б отвечает за сбор данных о пазлах, представленных на внешних площадках, таких как Amazon или eBay, с помощью библиотек requests и BeautifulSoup. Для каждого товара извлекаются: название, бренд, количество деталей (парсится из названия регулярным выражением, например по паттерну «1000 Piece»), цена, рейтинг, количество отзывов, тематика (определяется по ключевым словам в названии) и возрастная категория. Все собранные пазлы хранятся в классе PuzzleDataset, который обеспечивает дедупликацию по названию и поддерживает инкрементальное обновление. Результат — датасет реальных рыночных товаров для конкурентного и ценового анализа.

Задача Б2: Получение структурированных данных через API BoardGameGeek
BoardGameGeek (BGG) предоставляет бесплатный публичный XML API без необходимости регистрации или ключа. Участник Б реализует класс BGGScraper, который обращается к эндпоинту api.geekdo.com/xmlapi2 и получает данные о пазлах: пользовательские оценки сложности (по шкале от 1 до 5), рекомендуемый минимальный возраст, среднее время сборки, описательные теги и категории. Эти данные дополняют датасет с торговых площадок оценками сложности от сообщества, которых нет на Amazon. Например, можно проверить гипотезу: коррелируют ли оценки сложности из BGG с количеством деталей? Результаты объединяются с общим PuzzleDataset.

Задача Б3: Анализ спроса через Google Trends
С помощью библиотеки pytrends (неофициальный Python-клиент для Google Trends) участник Б собирает данные о динамике поискового интереса к пазлам. Анализируются два направления. Первое — сезонность: как меняется интерес к запросу «jigsaw puzzle» по месяцам за последние пять лет? Ожидаемый паттерн включает пики перед Рождеством и возможный всплеск в период локдаунов 2020–2021. Второе — региональная популярность: в каких странах и штатах США интерес к пазлам и конкретным брендам (Ravensburger, Trefl, Clementoni) наиболее высок? Это помогает бренду понять, на какие рынки стоит выходить.

Задача Б4: Рыночный анализ и статистические корреляции
Финальная задача участника Б — провести статистический анализ внешнего датасета пазлов. Исследуются три направления. Первое — зависимость цены от количества деталей: строится scatter plot и вычисляется коэффициент корреляции Пирсона, проверяется гипотеза о линейном росте цены с числом деталей при значимом влиянии бренда. Второе — зависимость рейтинга от цены: приводит ли более высокая цена к лучшим оценкам? Также анализируется метрика «цена за одну деталь» как показатель ценовой эффективности. Третье — популярность тематик: какие категории (животные, города, искусство, природа) наиболее представлены на рынке и получают наивысшие оценки в среднем. Этот анализ напрямую формирует рекомендации по производственной стратегии бренда.
