Проект: Анализ данных в индустрии пазлов

Разделение построено так, что один участник работает с внутренними бизнес-данными (профили пользователей и покупки с сайта бренда), а второй — с внешними рыночными данными (парсинг сторонних площадок и публичных API).

Дмитрий Ющенко — Внутренние данные и бизнес-метрики

Задача А1: Сбор и моделирование данных о пользователях
Проектирование структур данных, представляющих зарегистрированных пользователей сайта бренда. Сюда входят 

демографические данные: 
возраст или дата рождения, пол, страна и город проживания, язык интерфейса. 

поведенческие данные: 
дата регистрации, частота визитов (сессии), источник трафика (Определяется опросом). 

Основной результат — класс UserProfile, который хранит все эти атрибуты, включает вспомогательные методы, например вычисление возраста из даты рождения.

Задача А2: Сбор и моделирование данных о покупках и транзакциях
моделирование истории покупок каждого пользователя.

По каждой транзакции фиксируются: 
дата и время покупки, список приобретённых товаров (название пазла, количество деталей, тематика), итоговая сумма заказа, применённая скидка, использованный промокод, способ оплаты, регион доставки и статус доставки. 

Результат - класс Order, связанного с пользователем. Класс Order автоматически вычисляет итоговую сумму с учётом скидки. 

Задача А3: Вычисление бизнес-метрик по каждому пользователю

класс UserAnalytics, который вычисляет ключевые бизнес-метрики для каждого пользователя: 
LTV (Lifetime Value) — суммарная выручка с одного пользователя за всё время, позволяет выявить наиболее ценных клиентов. 
AOV (Average Order Value) — средний чек, используется для ценового позиционирования и оценки эффективности промоакций.
Churn — флаг, выставляемый пользователю, который не совершал покупок более 90 дней.
Retention показывает, какой процент пользователей вернулся за 30, 60 или 90 дней после первой покупки. 
Каждому пользователю также присваивается RFM-сегмент (Champion, Loyal, Promising, At Risk или Lost) на основе давности, частоты и суммы покупок.
Дополнительно вычисляются любимая тематика пазлов и предпочтительный уровень сложности (по среднему количеству деталей в заказах).

Задача А4: агрегированные метрики по всей базе

класс BusinessDashboard, который принимает полный список профилей пользователей и считает метрики на уровне всей компании: 
общая выручка, средний LTV, уровень оттока, разбивка пользователей по RFM-сегментам, топ тематик пазлов по популярности, выручка по городам и регионам.

Пальцева Ксения — Внешние рыночные данные и анализ трендов

Задача Б1: Парсинг товаров
Сбор данных о пазлах, представленных на внешних площадках.
Все собранные пазлы хранятся в классе PuzzleDataset, который обеспечивает дедупликацию по названию и поддерживает инкрементальное обновление. Результат — датасет реальных рыночных товаров для конкурентного и ценового анализа.

Задача Б2: Получение данных через API BoardGameGeek
BoardGameGeek (BGG) - бесплатный публичный XML API .

класс BGGScraper, который обращается к эндпоинту и получает данные о пазлах:
пользовательские оценки сложности, рекомендуемый минимальный возраст, среднее время сборки, описательные теги и категории.
Можно проверить гипотезу: коррелируют ли оценки сложности из BGG с количеством деталей? Результаты объединяются с общим PuzzleDataset.

Задача Б3: Анализ через Google Trends
С помощью pytrends собирать данные о динамике поискового интереса к пазлам. Анализируются два направления:
Первое — сезонность: как меняется интерес к запросу «jпазлы» по месяцам за последние пять лет?
Второе — региональная популярность: в каких регионах интерес к пазлам и конкретным брендам наиболее высок?

Задача Б4: Рыночный анализ и статистические корреляции
Статистический анализ внешнего датасета пазлов. Исследуются три направления:
Первое — зависимость цены от количества деталей: строится scatter plot и вычисляется коэффициент корреляции, проверяется гипотеза о линейном росте цены с числом деталей. 
Второе — зависимость рейтинга от цены: приводит ли более высокая цена к лучшим оценкам? Также анализируется метрика «цена за одну деталь» как показатель ценовой эффективности. 
Третье — популярность тематик: какие категории (животные, города, искусство, природа) наиболее представлены на рынке и получают наивысшие оценки в среднем.
